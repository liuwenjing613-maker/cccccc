## 1. 赛题背景与核心挑战

传统主动降噪（ANC）系统在平稳噪声下表现优异，但在**声学环境突变**（如佩戴者从安静室内突然走到嘈杂街道）时，容易出现系统发散或降噪性能骤降的问题。

本次大赛的核心目标是：**考察并提升深度神经网络（DNN）在面临未见过的噪声源、未见过的声学路径以及突变拼接噪声时的收敛能力与跟踪稳定性。**

为了极度贴近真实的物理硬件部署场景，组委会在数据集设计上做出了以下严格限制：

* **初级路径（Primary Path, $P(z)$）物理隐藏**：在真实场景下，算法无法提前预知外界噪声的传播路径。因此，本赛题**不提供**初级声学路径。。

##  2. 数据集概览与机制说明

请参赛者将下载好的官方数据集放置于项目根目录的 `dataset/` 文件夹下。

### 数据流拓扑与隐藏机制

在真实的 ANC 物理模型中，误差麦克风接收到的残余噪声 $e(n)$ 由下式决定：


$$e(n) = d(n) - \hat{d}(n) = [x(n) * P(z)] - [y(n) * S(z)]$$

为了隐藏 $P(z)$，组委会已在离线阶段计算好了**期望噪声 $d(n)$**。可获取的数据仅包含以下三部分：

| 数据类型 | 目录/文件 | 说明 |
| --- | --- | --- |
| **原始参考噪声** $x(n)$ | `dataset/NOISE/` | 参考麦克风采集的原始环境噪声（16-bit WAV，原生 48kHz）。 |
| **期望噪声** $d(n)$ | `dataset/EXPECTED_NOISE/` | 噪声经过未知初级路径衰减后到达误差麦克风的声音。**命名规则**：`{噪声名}_scene_{路径编号}.wav`。 |
| **次级路径** $S(z)$ | `dataset/sh_processed.npy` | 扬声器到误差麦克风的物理冲激响应（已做空间平均并降采样至 48kHz，尺寸为 `(Num_Paths, Length)`）。 |

### 突变测试集（Splicing Testing）

为考察网络在环境突变时的稳定性，基线的 Dataloader 在测试阶段（`is_train=False`）会自动抽取两种不同的测试噪声进行**硬拼接（Splicing）**。网络必须在 1 秒的片段内，应对突如其来的声学特征跃变，而不产生严重的伪影或发散。

##  3. 基线系统架构 (DeepANC-CRN)

官方提供的基线模型 `DeepANC_CRN` 采用带有因果卷积（Causal Convolution）的 U-Net 架构，辅以高效的编解码机制。

* **因果卷积（CausalConv2d）**：由于声学系统的实时性要求，网络决不能“预见”未来的声音。基线中的卷积层利用严格的时间轴非对称 Padding（`pad_time = kernel_size - 1`）保证了系统的物理因果性。
* **分组长短期记忆网络（GroupLSTM）**：在特征的瓶颈层（Bottleneck），基线引入了 GroupLSTM 以捕获长时时序依赖，分组操作显著降低了显存占用并提升了推理速度。
* **物理卷积对齐**：在模拟次级路径抵消（`apply_dynamic_path`）时，基线对一维卷积进行了**冲激响应时域翻转（Time-flip）**，完美对齐了 PyTorch 互相关（Cross-Correlation）与真实物理声学的前向卷积传播。

##  5. 评估指标与可视化

基线代码在每个测试场景结束后，会自动计算并输出**降噪量（Noise Reduction, NR）**，其定义为：


$$NR = 10 \log_{10} \left( \frac{\sum d(n)^2}{\sum e(n)^2 + \epsilon} \right)$$


*NR 值越大（单位 dB），表明网络的降噪性能越好。负值则代表网络导致了噪声放大。*

同时，基线会在根目录下自动生成两类极具分析价值的图表：

1. **时域波形图（Time Domain）**：展示初级噪声 $d(n)$ 与残余噪声 $e(n)$ 的前 900ms 波形对比，用于观察网络是否存在初始相位延迟。
2. **频域功率谱密度图（PSD）**：在对数频域上展示降噪前后的能量分布，帮助选手分析网络对高低频分量的处理能力。

## 6. 官方基线性能参考 (Baseline Performance Reference)

为了给参赛团队提供一个直观的性能对照，组委会使用官方提供的 `DeepANC_CRN` 模型在默认配置下进行了 **100 个 Epoch** 的训练。以下是该模型在突变拼接测试集上的降噪表现：

| 测试场景 (Scenario) | 考察维度 | 独立场景降噪量 (Scene NR) | 平均降噪量 (Avg NR) |
| --- | --- | --- | --- |
| **Test 1: 未见噪声 + 已见路径**<br>*(Unseen Noise, Seen Paths)* | 对未知环境噪声的泛化与突变跟踪能力 | `5.77`, `6.23`, `5.65`, `4.28`<br>`4.06`, `3.68`, `0.32`, `4.26` (dB) | **4.28 dB** |
| **Test 2: 未见噪声 + 未见路径**<br>*(Unseen Noise, Unseen Paths)* | 对全新物理空间与未知噪声的**双重极限泛化能力** | `4.69`, `1.07` (dB) | **2.88 dB** |

###  基线结果分析与赛题洞察

从上述基线测试结果中，我们可以得出以下几个关键结论，供参赛选手在优化算法时参考：

1. **算法闭环的可行性验证**：在隐藏了初级声学路径 $P(z)$ 的极其严苛的条件下，基线网络在所有测试场景中均取得了**正向的降噪量（NR > 0）**，证明了利用纯数据驱动方法隐式学习复杂声学规律的完全可行性。
2. **空间泛化衰减（核心痛点）**：对比 Test 1 和 Test 2 可以发现，当系统面对**完全未见过的物理路径**时，平均降噪量出现了明显的衰减（从 4.280 dB 下降至 2.88 dB）。如何提升网络跨物理空间的鲁棒性，是本赛题最大的挑战之一。
3. **场景方差显著**：在 Test 1 中，面对不同的拼接突变场景，网络的降噪表现波动极大（最高可达 6.23 dB，最低仅为 0.35 dB）。这表明基础的因果卷积与 LSTM 结构在某些特定频率突变或极端相位的处理上依然存在“盲区”。
